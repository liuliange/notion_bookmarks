# 小组件顶部新增「置顶推荐卡片」功能教程

## 一、功能概述

在网站顶部原有小组件区域**上方**，新增一个**置顶推荐卡片**区，用于展示需要在首页重点推广的内容。

所有推广内容在「导航链接」数据库中统一管理，通过打 `置顶推广` 标签控制哪些内容显示。

> 注意：置顶推荐卡片与原有小组件（时钟、天气等）是**两个独立区域，可同时显示**——顶部是推荐卡，下方仍是小组件。两者互不替代。

## 二、技术方案

| 项目 | 方案 |
|------|------|
| **数据来源** | 「导航链接」数据库，通过 `置顶推广` 标签过滤 |
| **角标配置** | 「网站配置」数据库新增 `WIDGET_BADGE_CONFIG` 属性 |
| **显示逻辑** | 推荐区：有带 `置顶推广` 标签的数据才显示；小组件区：`WIDGET_CONFIG` 非空才显示 |
| **角标渲染** | 在 `HomeWidgets` 内以绝对定位 `<span>` 浮层渲染于卡片右上角（**不修改 `LinkCard`**） |

## 三、涉及文件

| 文件路径 | 操作 | 说明 |
|----------|------|------|
| `src/app/api/promoted-links/route.ts` | **新建** | API 接口，获取带 `置顶推广` 标签的数据 |
| `src/components/widgets/HomeWidgets.tsx` | **修改** | 置顶卡片核心组件（含角标浮层） |
| `src/app/page.tsx` | **修改** | 移除 `WIDGET_CONFIG` 门控，让组件始终挂载 |
| `src/lib/notion.ts` | **修改** | 暴露 `WIDGET_BADGE_CONFIG` 配置 |
| `src/components/ui/LinkCard.tsx` | **未改动** | 角标不在 `LinkCard` 内渲染 |

## 四、Notion 配置步骤

### 4.1 网站配置数据库

新增一行配置：

| Name | Value |
|------|------|
| `WIDGET_BADGE_CONFIG` | `热销,#FF6B6B,上新,#4ECDC4,独家,#FFB347,限时,#A29BFE,推荐,#F39C12` |

**格式说明**：`角标文字1,颜色值1,角标文字2,颜色值2,...`（英文逗号分隔，按出现顺序对应第 1~5 张卡，最多 5 个）。

> 文字与颜色均可自定义，例如也可写成 `编辑推荐,#FF6B6B,人气精选,#4ECDC4,...`。

### 4.2 导航链接数据库

在需要显示在顶部的推广内容中，**Tags 字段**打上 `置顶推广` 标签。

> ⚠️ 添加标签后必须按 `Enter` 确认，标签变成蓝色气泡样式后保存。

## 五、代码文件

### 5.1 API 路由：`src/app/api/promoted-links/route.ts`

```typescript
// src/app/api/promoted-links/route.ts
import { getLinks } from '@/lib/notion';
import { NextResponse } from 'next/server';

// 推广广告位使用的标签，过滤后需从卡片中移除
const PROMOTED_TAG = '置顶推广';
const MAX_PROMOTED = 5;

export async function GET() {
  try {
    const allLinks = await getLinks();

    const promotedLinks = allLinks
      .filter((link) => link.tags?.includes(PROMOTED_TAG))
      .map((link) => ({
        ...link,
        // 移除推广标签，避免显示在卡片底部
        tags: link.tags.filter((tag) => tag !== PROMOTED_TAG),
      }))
      .slice(0, MAX_PROMOTED);

    // 注意：返回结构为 { links: [...] }，无 success 字段
    return NextResponse.json({ links: promotedLinks });
  } catch (error) {
    console.error('获取推广链接失败:', error);
    return NextResponse.json({ links: [] });
  }
}
```

### 5.2 小组件组件：`src/components/widgets/HomeWidgets.tsx`

```tsx
'use client';

import React, { useEffect, useState } from 'react';
import dynamic from 'next/dynamic';
import { WidgetSkeleton, HotNewsSkeleton } from '@/components/ui/Skeleton';
import { WebsiteConfig, Link } from '@/types';
import { motion } from 'framer-motion';
import { useTheme } from 'next-themes';
import LinkCard from '@/components/ui/LinkCard';

// 小组件用动态导入（带骨架屏占位）
const SimpleTime = dynamic(() => import('@/components/widgets/SimpleTime'), {
  loading: () => <WidgetSkeleton className="w-[280px] h-[150px]" />, ssr: false,
});
const AnalogClock = dynamic(() => import('@/components/widgets/AnalogClock'), {
  loading: () => <WidgetSkeleton className="w-[150px] h-[150px]" />, ssr: false,
});
const Weather = dynamic(() => import('@/components/widgets/Weather'), {
  loading: () => <WidgetSkeleton className="w-[400px] h-[150px]" />, ssr: false,
});
const IPInfo = dynamic(() => import('@/components/widgets/IPInfo'), {
  loading: () => <WidgetSkeleton className="w-[220px] h-[150px]" />, ssr: false,
});
const HotNews = dynamic(() => import('@/components/widgets/HotNews'), {
  loading: () => <HotNewsSkeleton className="w-[300px] h-[150px]" />, ssr: false,
});
const StockWatchlist = dynamic(() => import('@/components/widgets/StockWatchlist'), {
  loading: () => <WidgetSkeleton className="w-[340px] h-[150px]" />, ssr: false,
});

interface HomeWidgetsProps {
  config: WebsiteConfig;
}

// 解析角标配置：热销,#FF6B6B,上新,#4ECDC4,... -> [{label,color}, ...]
function parseBadgeConfig(raw?: string): { label: string; color: string }[] {
  if (!raw) return [];
  const parts = raw.split(',').map((s) => s.trim()).filter(Boolean);
  const badges: { label: string; color: string }[] = [];
  for (let i = 0; i + 1 < parts.length; i += 2) {
    badges.push({ label: parts[i], color: parts[i + 1] });
  }
  return badges;
}

export default function HomeWidgets({ config }: HomeWidgetsProps) {
  const widgetMap: Record<string, React.ReactNode> = {
    '简易时钟': <SimpleTime />,
    '圆形时钟': <AnalogClock />,
    '天气': <Weather />,
    'IP信息': <IPInfo />,
    '热搜': <HotNews />,
    '自选股': <StockWatchlist />,
  };

  const widgetConfig = config.WIDGET_CONFIG?.split(',').map(s => s.trim()).filter(Boolean) ?? [];
  const badgeConfig = parseBadgeConfig(config.WIDGET_BADGE_CONFIG);

  // 推广广告位数据
  const [promotedLinks, setPromotedLinks] = useState<Link[]>([]);
  const [loadingPromoted, setLoadingPromoted] = useState(true);

  useEffect(() => {
    let cancelled = false;
    fetch('/api/promoted-links')
      .then((res) => res.json())
      .then((data) => { if (!cancelled) setPromotedLinks(data.links ?? []); })
      .catch(() => { if (!cancelled) setPromotedLinks([]); })
      .finally(() => { if (!cancelled) setLoadingPromoted(false); });
    return () => { cancelled = true; };
  }, []);

  const hasPromoted = !loadingPromoted && promotedLinks.length > 0;
  const { theme } = useTheme();
  const isMacintosh = theme?.includes('macintosh');

  return (
    <>
      {/* 推广广告位：有数据才显示；仅桌面端（md+）显示，间距对齐 LinkContainer */}
      {hasPromoted && (
        <div className="hidden md:block w-full mb-8 px-4">
          <motion.div
            initial={{ opacity: 0, y: -10 }}
            animate={{ opacity: 1, y: 0 }}
            transition={{ duration: 0.5 }}
            className="relative"
          >
            <div className="flex gap-3 overflow-x-auto pb-4 scrollbar-none">
              {promotedLinks.map((link, idx) => {
                const badge = badgeConfig[idx]; // 配置缺失则该卡不显示角标
                return (
                  <div key={link.id} className="relative flex-1 min-w-[180px] max-w-[220px]">
                    {/* 右上角贴边角标，不占用标题空间；麦金塔主题下移避免遮挡标题栏装饰 */}
                    {badge && (
                      <span
                        className={`absolute ${isMacintosh ? 'top-5' : 'top-2'} right-2 z-10 rounded-full px-1.5 py-0.5 text-[10px] font-semibold text-white pointer-events-none`}
                        style={{ backgroundColor: badge.color }}
                      >
                        {badge.label}
                      </span>
                    )}
                    <LinkCard link={link} />
                  </div>
                );
              })}
            </div>
          </motion.div>
        </div>
      )}

      {/* 小组件区：WIDGET_CONFIG 非空才显示，与推广区相互独立 */}
      {widgetConfig.length > 0 && (
        <div className="hidden lg:block w-full mb-8">
          <motion.div
            initial={{ opacity: 0, y: -10 }}
            animate={{ opacity: 1, y: 0 }}
            transition={{ duration: 0.5 }}
            className="relative overflow-hidden"
          >
            <div className="flex overflow-x-auto pb-4 scrollbar-thin scrollbar-thumb-primary/20 scrollbar-track-transparent">
              <div className="flex space-x-4 px-1">
                {widgetConfig.map((name, idx) => {
                  const Comp = widgetMap[name];
                  if (!Comp) return null;
                  return (
                    <div key={name + '-' + idx} className="flex-shrink-0">
                      {Comp}
                    </div>
                  );
                })}
              </div>
            </div>
          </motion.div>
        </div>
      )}
    </>
  );
}
```

> 说明：`LinkCard` 无需任何改动，角标作为浮层覆盖在卡片右上角，不挤占标题行，因此标题与外链图标都不受影响。

### 5.3 页面入口：`src/app/page.tsx`

移除 `WIDGET_CONFIG` 门控，让 `HomeWidgets` 始终挂载（内部自行判断两个区域的显隐）：

```tsx

  <div className="w-full">
    <HomeWidgets config={config} />
  </div>

```

## 六、关键技术要点

| 技术点 | 实现方式 | 说明 |
|--------|----------|------|
| **数据获取** | API 路由 `/api/promoted-links` | 服务端取数，前端 `fetch` 后取 `data.links` |
| **标签过滤** | `tags.includes('置顶推广')` | 仅带此标签的链接出现在顶部 |
| **标签隐藏** | 返回时从 `tags` 数组移除 `置顶推广` | 避免该标签显示在卡片底部 |
| **角标配置** | `WIDGET_BADGE_CONFIG` | 自定义文字和颜色，格式：`文字1,颜色1,文字2,颜色2,...` |
| **角标渲染** | 绝对定位 `<span>` 浮层（在 `HomeWidgets` 内） | 不改动 `LinkCard`，不占用标题行 |
| **角标兜底** | 配置缺失的索引**不显示角标** | 不会用默认「推荐」填充 |
| **主题适配** | 麦金塔主题 `top-5`，其余 `top-2` | 避开标题栏装饰 |
| **移动端隐藏** | 推荐区 `hidden md:block` | 移动端不显示推荐卡 |
| **隐藏滚动条** | `scrollbar-none` | 保留横向滚动能力，但不显示滚动条 |

## 七、日常使用流程

1. 在「导航链接」数据库添加推广内容（名称、描述、图标、链接等）
2. 在 Tags 字段中精确添加 `置顶推广` 标签（按 Enter 确认）
3. 顶部自动显示推荐卡片，右上角显示对应角标
4. 调整角标文字/颜色：修改网站配置中的 `WIDGET_BADGE_CONFIG`
5. 最多显示 5 张卡，取前 5 条带 `置顶推广` 标签的数据
6. 若无任何数据带 `置顶推广` 标签，则顶部推荐区不显示，但下方小组件照常显示

## 八、配置说明

### 默认角标配置（示例）

| 位置 | 角标文字 | 颜色值 |
|------|----------|--------|
| 第 1 个 | 热销 | `#FF6B6B` |
| 第 2 个 | 上新 | `#4ECDC4` |
| 第 3 个 | 独家 | `#FFB347` |
| 第 4 个 | 限时 | `#A29BFE` |
| 第 5 个 | 推荐 | `#F39C12` |

### 自定义角标

修改网站配置中的 `WIDGET_BADGE_CONFIG` 值即可，格式为：

```
文字1,颜色值1,文字2,颜色值2,文字3,颜色值3,文字4,颜色值4,文字5,颜色值5
```

角标按数组顺序依次对应第 1~5 张卡；若某位置未配置，则该卡不显示角标。

## 九、常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 推荐卡完全不显示 | 没有带 `置顶推广` 标签的数据 | 在导航链接中给记录打标签并按 Enter 确认 |
| 角标不显示 | `WIDGET_BADGE_CONFIG` 未配置或该位置缺值 | 按格式填写角标配置（成对出现） |
| 只有部分卡片显示 | 标签未精确匹配 / 数据不足 5 条 | 复制已成功的标签值粘贴；不足 5 条不补位 |
| 卡片显示但标签出现在底部 | `置顶推广` 标签未被过滤 | 检查 API 是否从 `tags` 中移除了该标签 |
| 麦金塔角标位置不对 | 主题特殊装饰 | 调整 `isMacintosh ? 'top-5' : 'top-2'` 中的值 |
| 小组件也不显示 | `WIDGET_CONFIG` 为空 | 在网站配置中填写小组件名称 |
